## 2. PHASE 1 — AVRIL : BUILD IN PUBLIC E-COM + DONNÉES TERRAIN CDV

### 2.1 Objectif

Installer la présence F2 comme studio AI agents, publier les données terrain CDV (530+ reviews, Reddit, Mastercard), et documenter la construction des 3 premiers SaaS e-com en build in public.

### 2.2 Semaine 1 — Lancement

| Jour | Pilier squelette | Contenu spécifique semaine 1 |
|------|-----------------|------------------------------|
| **Lundi** | Metrics Portfolio | Tweet d'annonce : qui on est, ce qu'on build (6 AI agents, 3 verticales), pourquoi. Pas de métriques encore. Lien vers foundrytwo.com en reply. |
| **Mardi** | Hot Take Studio | Première donnée terrain CDV : "$800/month — the average chargeback cost Shopify merchants don't track." |
| **Mercredi** | Data Thread | Premier thread CDV : 530+ reviews analysées, 71% friendly fraud, apps JS, ghost billing. |
| **Jeudi** | Communauté | Question ouverte : "Shopify merchants: what's your biggest hidden cost running your store?" |
| **Vendredi** | Build in Public | Recap semaine 1 : portfolio status (3 SaaS en build, 105 features defined, 10 anti-competitor principles). |

Cross-engagement R+F sur chaque post : obligatoire dès le jour 1. Le protocole doit être rodé dès la semaine 1 (cf. twitter/context.md §2).

### 2.3 Semaine 2 — Installation de la routine

| Jour | Pilier squelette | Contenu spécifique semaine 2 |
|------|-----------------|------------------------------|
| **Lundi** | Metrics Portfolio | Premiers vrais chiffres : signups semaine 1 (tous SaaS confondus), MRR total, verticals actives. |
| **Mardi** | Hot Take Studio | Hot take basé sur données CDV : "71% of chargebacks are friendly fraud. The customer got the product. They just... disputed anyway." |
| **Mercredi** | Data Thread | Thread terrain : apps injecting JS dans les stores, ghost billing ($127/month en apps inutilisées). |
| **Jeudi** | Communauté | Poll : "What's the biggest hidden cost for e-com merchants?" (chargebacks / unused apps / return fees / processor markups). |
| **Vendredi** | Build in Public | Recap semaine 2 + comparaison semaine 1 vs semaine 2. Progression features shipped. |

**Objectif fin de semaine 2 :** le cycle batch samedi → publication semaine tourne sans friction. Si le batch prend plus de 2h pour les 5 posts F2, identifier le goulot et optimiser.

### 2.4 Semaines 3-4 — Store Insight of the Week

| Évolution | Détail |
|-----------|--------|
| **"Store Insight of the Week"** | 1 insight terrain e-com présenté en détail (données CDV, pattern marchand, coût caché). Remplace le data thread du mercredi 1 semaine sur 2. |
| **Data threads plus denses** | Les données CDV terrain sont suffisamment riches dès le départ (530+ reviews). Les threads s'enrichissent avec de nouvelles sources (forums, Stripe data, etc.). |
| **Hot takes data-driven** | Les hot takes du mardi s'appuient sur des données CDV terrain précises, pas des observations générales. |

### 2.5 Milestones Phase 1

| Milestone | Cible | Si pas atteint |
|-----------|-------|----------------|
| Profil F2 100% complet + Premium actif | Avant premier post | Ne pas lancer sans Premium — les liens seront invisibles. |
| Cross-engagement complété sur 100% des posts | Fin semaine 1 | Revoir le protocole avec F. Identifier le blocage (horaire ? oubli ? process pas clair ?). |
| 20-50 followers F2 | Fin avril | Normal si < 50. Vérifier que les cross-engagements de R et F amènent des clics profil. |
| Batch samedi < 2h pour F2 | Fin semaine 2 | Optimiser les prompts Claude, préparer les données en amont. |
| 5+ replies organiques (hors R et F) reçues | Fin avril | Le contenu ne provoque pas assez de réaction. Revoir les hooks et les questions de fin de tweet. |
| 1+ Store Insight of the Week publié | Fin avril | Si les données CDV ne génèrent pas assez d'engagement, tester un angle différent (comparaison avant/après, calcul de perte mensuelle). |

## 3. PHASE 2 — MAI : LANCEMENTS + MULTI-VERTICAL

### 3.1 Objectif

Lancer les premiers SaaS, élargir le contenu aux 3 verticales (e-com, agencies, creators), et commencer à collecter des données terrain sur les nouvelles verticales.

### 3.2 Semaines 5-6 — Premiers lancements

| Évolution | Détail |
|-----------|--------|
| **Annonces de lancement** | Chaque lancement SaaS = 1 thread dédié (remplace le vendredi Build in Public de la semaine). |
| **Métriques multi-produit** | Le lundi Metrics couvre désormais plusieurs SaaS : MRR total, signups par produit, verticals actives. |
| **Premières mentions de résultats** | Si premiers clients payants : le mentionner dans le recap vendredi. Chiffre exact, pas de hype. |

### 3.3 Semaines 7-8 — Contenu multi-vertical

| Évolution | Détail | Conditions d'activation |
|-----------|--------|------------------------|
| **"Agent Insight of the Week"** | Élargissement du "Store Insight of the Week" aux 3 verticales. 1 insight par semaine, rotation e-com / agencies / creators. Remplace le data thread du mercredi 1 semaine sur 2. | Au moins 2 SaaS lancés dans des verticales différentes. |
| **Premiers témoignages** | Intégrer dans le recap vendredi un quote d'utilisateur (avec permission). | Au moins 1 retour positif reçu en DM ou reply. |
| **Thread "What we learned"** | Thread récap mensuel (fin avril et fin mai) : les top learnings, les top patterns, les surprises. | Données suffisantes pour un thread de 5-6 tweets. |

### 3.4 Planning type semaine Phase 2

Le squelette reste identique (cf. f2/context.md §4). Les changements sont dans la **densité** et la **diversité verticale** du contenu :

| Jour | Évolution vs Phase 1 |
|------|---------------------|
| Lundi (Metrics Portfolio) | Métriques multi-SaaS, comparaisons semaine N vs N-1, répartition par verticale. |
| Mardi (Hot Take) | Insights issus de données terrain multi-verticales (e-com + agencies + creators). |
| Mercredi (Data / Agent Insight) | Alternance : semaine A = data thread agrégé, semaine B = Agent Insight of the Week. |
| Jeudi (Communauté) | Questions ciblées par verticale en rotation : merchants → freelancers → creators. |
| Vendredi (Build in Public) | Inclut témoignages, progression portfolio, MRR total, et annonces de lancement le cas échéant. |

### 3.5 Milestones Phase 2

| Milestone | Cible | Si pas atteint |
|-----------|-------|----------------|
| 100-300 followers F2 | Fin mai | Analyser : les RT de R et F amènent-ils des follows ? Si non, revoir le contenu F2 (valeur insuffisante pour follow). |
| Engagement rate > 1% | Fin mai | Revoir les hooks. Tester des formats différents (plus de polls, plus de questions directes). |
| 10-30 clicks UTM/semaine vers produits portfolio | Fin mai | Vérifier que le lien en reply est bien posté systématiquement. Tester des CTA différents. |
| 2+ SaaS lancés | Fin mai | Si retard de lancement : maintenir le contenu terrain CDV + build in public, ajuster le planning. |
| Replies organiques > cross-engagement | Mi-mai | Seuil clé : les réponses d'inconnus dépassent les réponses R+F. La communauté prend le relais. |
| Batch samedi < 1h30 pour F2 | Fin mai | Le process doit être rodé. Si encore > 2h : trop de friction dans les prompts ou les données. |

## 4. PHASE 3 — JUIN+ : FULL PORTFOLIO + AUTORITÉ STUDIO

### 4.1 Objectif

Positionner F2 comme le studio AI agents de référence pour e-com, agencies et creators. Le portfolio est suffisamment large pour un contenu riche et diversifié.

### 4.2 Semaines 9-12 — UGC et benchmarks

| Évolution | Détail | Conditions d'activation |
|-----------|--------|------------------------|
| **Community Spotlight** | Mettre en avant un utilisateur et ses résultats 1x/semaine. Remplace le jeudi communauté 1 semaine sur 2. | Au moins 3 utilisateurs actifs qui partagent leurs résultats. |
| **Benchmarks par verticale** | Data thread avec comparaisons par verticale (e-com vs agencies vs creators) et par SaaS. | Données d'usage suffisantes sur 3+ SaaS. |
| **Contenu issu de la communauté** | Reposter (via quote tweet) les résultats partagés par des utilisateurs. | Utilisateurs qui mentionnent @foundrytwo ou un produit du portfolio dans leurs tweets. |

### 4.3 Semaines 13+ — Autorité installée

| Évolution | Détail | Conditions d'activation |
|-----------|--------|------------------------|
| **Guest expert threads** | Inviter un expert e-com/agency/creator à co-analyser des données en thread. F2 publie, l'expert est taggé. | Relation établie avec au moins 1 expert Tier 1 ou Tier 2 qui a déjà engagé avec F2. |
| **Série récurrente identifiable** | Le "Agent Insight of the Week" ou le data thread du mercredi devient un rendez-vous reconnu. Les gens l'attendent. | Replies spontanées type "What's this week's insight?" ou "Do my store next". |
| **Vision portfolio complète** | Les 6 SaaS sont mentionnés régulièrement. F2 = FoundryTwo le studio, pas un seul produit. | Au moins 5 SaaS lancés ou en beta publique. |

### 4.4 Planning type semaine Phase 3

| Jour | Évolution vs Phase 2 |
|------|---------------------|
| Lundi (Metrics Portfolio) | Dashboard mensuel, tendances sur 2+ mois, MRR total, croissance par verticale. |
| Mardi (Hot Take) | Positions plus affirmées — F2 a la data pour les soutenir. Insights cross-verticales. Toujours sans agressivité. |
| Mercredi (Data / Agent Insight) | Threads plus ambitieux : benchmarks cross-verticales, comparaisons sectorielles, séries thématiques. |
| Jeudi (Communauté / Spotlight) | Alternance : semaine A = question/poll, semaine B = community spotlight. |
| Vendredi (Build in Public) | Vision portfolio complète. Roadmap publique des 6 SaaS. Progression mensuelle. |

### 4.5 Milestones Phase 3

| Milestone | Cible | Si pas atteint |
|-----------|-------|----------------|
| 300-1000 followers F2 | Fin juillet | Si < 300 : problème structurel. Revoir entièrement la stratégie contenu F2 ou envisager de fusionner l'effort sur les comptes perso uniquement. |
| Engagement rate > 2% | Fin juillet | Si stagne à 1% : le contenu est vu mais ne provoque pas de réaction. Augmenter la fréquence des polls et des calls to action directs. |
| 30-80 clicks UTM/semaine vers produits portfolio | Fin juillet | Si < 30 : les CTA en reply ne convertissent pas. Tester de nouveaux formats de lien (thread dédié, tweet single avec CTA fort). |
| DMs entrants 5+/semaine | Fin juillet | Si < 5 : la confiance n'est pas encore là. Le contenu informe mais ne crée pas assez de connexion. Plus de storytelling, plus de vulnérabilité. |
| 1+ community spotlight publié | Mi-juin | Si aucun utilisateur ne partage ses résultats : créer une incitation (mention, accès gratuit en échange d'un témoignage public). |
| Guest expert thread réalisé | Fin juillet | Si aucun expert n'a accepté : la relation n'est pas assez mûre. Augmenter l'engagement de R sur les comptes Tier 1. |

## 5. OPTIMISATION DU PROFIL F2

Le profil évolue avec les milestones du portfolio. Chaque mise à jour est une occasion de communication ("We just updated our bio because [milestone]").

| Quand | Élément | Mise à jour |
|-------|---------|-------------|
| **Avant premier post** | Bio | Version de lancement (cf. f2/context.md §1). |
| **Premier client payant** | Bio | Mentionner le premier revenu réel. |
| **100 signups portfolio** | Bio | Ajouter "100+ signups across the portfolio" ou équivalent. |
| **MRR > $100** | Bio | Mentionner le MRR ou le nombre de clients payants. |
| **2ème SaaS lancé** | Bio + lien | Ajouter la mention du second produit. Lien vers foundrytwo.com (hub portfolio). |
| **3+ SaaS lancés** | Nom affiché | Évaluer l'ajout d'un descripteur : "FoundryTwo · AI Agent Studio" ou "FoundryTwo · 6 AI Agents". |
| **Mi-mai** | Banner | Évaluer si la banner actuelle fonctionne. Si le profil convertit mal (visits vs follows < 10%), tester une banner avec le portfolio visuel ou un chiffre clé. |

## 6. CHECKPOINTS DE DÉCISION

Points fixes où R et F évaluent la performance F2 et prennent des décisions structurelles.

### Fin semaine 2 — Premier checkpoint

| Question | Si oui | Si non |
|----------|--------|--------|
| Le batch samedi tourne sans friction ? | Continuer. | Identifier le goulot. Simplifier le process. |
| Le cross-engagement est fait sur 100% des posts ? | Continuer. | Revoir l'organisation avec F (horaires, rappels). |
| Au moins 5 replies organiques reçues ? | Le contenu accroche. Continuer la trajectoire. | Retravailler les hooks et les questions de clôture des tweets. |

### Fin avril — Deuxième checkpoint

| Question | Si oui | Si non |
|----------|--------|--------|
| Engagement rate > 0.5% ? | Trajectoire correcte. Préparer le contenu multi-vertical pour mai. | Tester plus de polls et de questions directes. Analyser quels jours du squelette performent le moins. |
| Store Insight of the Week génère de l'engagement ? | Continuer le format, préparer l'élargissement en "Agent Insight". | Tester un angle différent : calcul de perte personnalisé, comparaison par secteur. |
| Followers F2 > 30 ? | Croissance organique installée. | Vérifier que R et F RT systématiquement + que le profil F2 est complet et attractif. |

### Fin mai — Troisième checkpoint

| Question | Si oui | Si non |
|----------|--------|--------|
| Replies organiques > replies R+F ? | La communauté prend le relais. Réduire le cross-engagement à 80%. | Maintenir le cross-engagement à 100%. Augmenter l'engagement de R sur les comptes Tier 1-2 pour attirer plus de monde vers F2. |
| 10+ clicks UTM/semaine vers produits portfolio ? | Le funnel Twitter → site fonctionne. | Revoir les CTA en reply. Tester "Free trial — link below" vs "Full breakdown here". |
| Batch < 1h30 ? | Le système est rodé. Focus sur la qualité du contenu. | Audit du workflow : quel poste prend le plus de temps ? Améliorer les prompts Claude ou simplifier les visuels. |

### Fin juin — Quatrième checkpoint (prépare Phase 4)

| Question | Si oui | Si non |
|----------|--------|--------|
| 300+ followers ? | F2 a une audience. Planifier Phase 4 : collabs, guest content, élargissement thématique. | Décision structurelle : F2 continue tel quel, ou on concentre l'effort sur R et F uniquement ? |
| Guest expert thread faisable ? | Planifier pour juillet. | Pas assez de relations Tier 1. R doit intensifier l'engagement niche 1 mois de plus. |
| Le contenu F2 reflète-t-il le studio multi-produit ? | Les updates F2 couvrent le portfolio complet (6 SaaS, 3 verticales). F2 = FoundryTwo le studio AI agents. | Si peu de SaaS lancés : maintenir le positionnement studio mais focus sur les produits live + données terrain. |
