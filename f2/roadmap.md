| **Process batch rodé** | Le batch vendredi/samedi est en place depuis le lancement LD. Le cross-engagement est un réflexe. |

| Aspect | Détail |
|--------|--------|

| **Squelette éditorial** | Lundi metrics portfolio. Mardi hot take e-com (basé sur données terrain). Mercredi data thread CDV. Jeudi questions merchants. Vendredi build in public portfolio. |
| **IH** | Build in public portfolio (pas mono-produit). Show IH à chaque lancement de SaaS. |

## 5. PHASE 3 — JUIN+ 2026 : AUTORITÉ STUDIO + PORTFOLIO COMPLET

### 5.1 Ce qui change

| Changement | Impact sur la gestion F2 |
|-----------|-------------------------|
| **Portfolio complet (6 SaaS)** | Contenu riche en données cross-produit et cross-vertical. |
| **Positionnement "AI agent studio"** | La cadence de production ET la couverture multi-vertical SONT le contenu. |
| **Communauté organique** | Merchants, freelancers, creators reply sans que R/F aient engagé en premier. |
| **Guides cross-produit sur IH** | Assez de données pour publier des guides actionnables ("How we build 3 AI agents per month", "What 530+ app reviews taught us about Shopify merchants"). |
| **Scale** | Le process est rodé, le batch est un réflexe, la communauté se nourrit elle-même. |

### 5.2 Milestones Phase 3

| Milestone | Cible | Si pas atteint |
|-----------|-------|----------------|
| @foundrytwo est identifié comme un STUDIO multi-vertical (pas un seul produit) | Fin juin | Accentuer la narration studio : chaque post rappelle le portfolio et les 3 verticals. |
| 1+ guide cross-produit publié sur IH | Fin juin | La matière est là (6 produits). Bloquer 30 min au batch pour le produire. |
| Des gens mentionnent F2 organiquement | Fin juin | L'engagement communautaire (R et F) n'est pas assez fort. Intensifier. |
| Chaque lancement PH performe mieux que le précédent | Continu | Analyser au cas par cas : produit moins attractif ou process qui régresse ? |
| Le temps total F2 pour R est stabilisé et soutenable | Fin juin | Si R est surchargé : identifier ce qui prend trop de temps. Simplifier ou automatiser. |
